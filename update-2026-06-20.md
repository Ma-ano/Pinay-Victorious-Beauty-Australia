# Update 2026-06-20
## Features
- **Removed featured brand section from below Best Selling** — featured brand slides now only appear in HeroBanner, no duplicate section beneath Best Selling carousel
- **Body scroll lock on product modal** — when admin product form is open, `document.body.style.overflow` is set to `"hidden"` to prevent background scrolling; restored on close/unmount

## Bug fixes
- **Fixed "Function setDoc() called with invalid data. Unsupported field value: undefined"** — added `stripUndefined()` helper in `product-store.ts` that strips `undefined` values from the object before passing to Firestore's `setDoc`. This affected `discount` and `originalPrice` fields when left empty in the admin form (evaluated to `undefined` by the ternary)
- **Fixed new products not appearing on customer-facing pages** — `getAllProducts()` in `product-store.ts` now provides defaults for fields that `saveProduct` intentionally omits (auto-calculated): `rating: 0`, `reviews: 0`, `sold: 0`, `images: []`, `variants: []`. Previously these were `undefined` at runtime on customer pages like Shop, Home, and Sale, causing `Math.floor(undefined)` → `NaN` in `ProductCard.tsx` and rendering `undefined` text in the UI

## Known issues
